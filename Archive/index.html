Stuff I've been doing

I've been learning Python for a while now and have come across the library pyautogui thanks to Al Sweigart's Automate The Boring Stuff with Python. This was the most interesting library I've come across so far and I decided to do some stuff with it. Nothing much, all simple stuff, but it was quite a challenge to me, especially since I've never coded something longer than 200 lines before.

This here is a autoclicker for the game Cookie Clicker



And this here is an autoclicker for the game SushiGoRound (RIP ADOBE FLASH). This project was magnitudes harder than the previous one as I had to analyse the images, keep track of ingredients, clear plates, compile recipes at the same time, so a lot more functions were needed.

I experimented with a Raspberry Pi 3 I had at home to try to send data from the Raspberry Pi over the internet. I've learnt how to use SSH to connect my pc to control linux computers (my RPi here) from my main laptop with PuTTY and VNC Viewer. Here, I used a simple circuit to test whether two wires are connected.


I used a python script on my Raspberry Pi as well as a PHP script on the database to send 'Connected' or 'Not Connected' to the database based on whether the wires are connected. If there is no change, nothing is sent. This uses the $_GET superglobal variable to grab data from HTML links. For example, in this website, https://www.randomwebsite.com/script.php?variable=value the 'variable' with value of 'value' will be saved into $_GET and we can access the variable. This is then sent to the database with SQL through another php script.

This is the circuit board I created. It's extremely simple with just 2 resistors and 5 connecting pins. I was thinking of making it into a lock-checking system (if is unlocked, last value sent to database would be not connected) but did not actually go ahead with it. Unfortunately, there would be no further continuation of this project.
